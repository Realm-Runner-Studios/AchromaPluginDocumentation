\doxysubsection{Detailed Description}
\doxylink{class_uni_web_view}{Uni\+Web\+View} Safe Browsing provides a way for browsing the web content in a more browser-\/like way, such as Safari on i\+OS and Chrome on Android. 

This class wraps {\ttfamily SFSafari\+View\+Controller} on i\+OS and "{}\+Custom Tabs"{} on Android. It shares cookies, auto-\/fill completion and other more data with the browser on device. Most of permissions are also built-\/in supported. You can use this class for some tasks that are limited for a normal web view, such as using Apple Pay or Progressive Web Apps (PWA).

You create a {\ttfamily \doxylink{class_uni_web_view_safe_browsing}{Uni\+Web\+View\+Safe\+Browsing}} instance by calling the static {\ttfamily \doxylink{class_uni_web_view_safe_browsing_a9d5991e3234fd927a80748b1b11bf35f}{Uni\+Web\+View\+Safe\+Browsing.\+Create}} method with a destination URL. You cannot change this URL once the instance is created. To show the safe browsing, call {\ttfamily Show} on the instance. The web content will be displayed in full screen with a toolbar containing the loaded URL, as well as some basic controls like Go Back, Go Forward and Done.

Browsing web content in {\ttfamily \doxylink{class_uni_web_view_safe_browsing}{Uni\+Web\+View\+Safe\+Browsing}} is only supported on i\+OS and Android. There is no such component in Unity Editor. Creating and showing a {\ttfamily \doxylink{class_uni_web_view_safe_browsing}{Uni\+Web\+View\+Safe\+Browsing}} on Unity Editor will fall back to open the URL in external browser by using Unity\textquotesingle{}s {\ttfamily Application.\+Open\+URL}.

{\footnotesize\ttfamily bool Uni\+Web\+View\+Safe\+Browsing.\+Is\+Safe\+Browsing\+Supported\hspace{0.3cm}{\ttfamily [static]}, {\ttfamily [get]}}



Whether the safe browsing mode is supported in current runtime or not. 

If supported, the safe browsing mode will be used when {\ttfamily Show} is called on a {\ttfamily \doxylink{class_uni_web_view_safe_browsing}{Uni\+Web\+View\+Safe\+Browsing}} instance. Otherwise, the system default browser will be used to open the page when {\ttfamily Show} is called.

This property always returns {\ttfamily true} on i\+OS runtime platform. On Android, it depends on whether there is an Intent can handle the safe browsing request. Usually it is provided by Chrome. If there is no Intent can open the URL in safe browsing mode, this property will return {\ttfamily false}.

To use this API on Android when you set your Target SDK to Android 11 or later, you need to declare the correct intent query explicitly in your Android\+Manifest.\+xml, to follow the Package Visibility (\href{https://developer.android.com/about/versions/11/privacy/package-visibility}{\texttt{ https\+://developer.\+android.\+com/about/versions/11/privacy/package-\/visibility}})\+:
