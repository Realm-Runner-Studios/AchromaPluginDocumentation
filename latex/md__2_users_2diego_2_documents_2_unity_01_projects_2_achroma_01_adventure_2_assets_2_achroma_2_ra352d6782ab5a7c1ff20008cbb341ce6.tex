Due to introduction of i\+Phone 6 Plus and its new @3x scale, the old way would not work well. Now, all screen-\/size-\/based insets calculation for \doxylink{class_uni_web_view}{Uni\+Web\+View} should NOT use Running\+On\+Retina\+IOS anymore. Instead you should use Uni\+Web\+View\+Helper.\+screen\+Height and Uni\+Web\+View\+Helper.\+screen\+Width and specify the point directly. For example, if you want to show a web view taking the upper half of screen, before version 1.\+7.\+0, it might be something like this\+: \begin{DoxyVerb}int uiFactor = UniWebViewHelper.RunningOnRetinaIOS() ? 2 : 1;

_webView.insets = new UniWebViewEdgeInsets(0,0,bottomInset,0);
\end{DoxyVerb}
 Right now, with the new API, you can just use this\+: \begin{DoxyVerb}int bottomInset = (int)(UniWebViewHelper.screenHeight * 0.5f);
_webView.insets = new UniWebViewEdgeInsets(0,0,bottomInset,0);
\end{DoxyVerb}
 No more ui factor and device-\/specified calculation needed.

To make sure you can notice this change, the usage of Running\+On\+Retina\+IOS() will cause an error now. You can just delete all usage of that, and change your insets code like the example above.
