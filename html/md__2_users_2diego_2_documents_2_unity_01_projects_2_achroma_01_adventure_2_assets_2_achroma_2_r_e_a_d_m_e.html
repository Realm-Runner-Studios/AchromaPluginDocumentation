Achroma Plugin#

1. Install dependencies

First of all, under Build Settings, choose Android platform

  1. Create an empty project
  2. Download firebase from the link: https://dl.google.com/firebase/sdk/unity/firebase_unity_sdk_9.6.0.zip
  3. Decompress the zip file
  4. From the folder dotNet4, install the following unity packages:
    • FirebaseAuth
    • FirebaseDatabase
    • FirebaseFunctions
    • FirebaseMessaging
    • FirebaseRemoteConfig
  5. Copy the Firebase config files "google-services.json" and "GoogleService-Info.plist" to the Assets/Firebase folder
  6. Upgrade the External Dependency Manager by downloading the package from https://github.com/googlesamples/unity-jar-resolver/blob/master/external-dependency-manager-latest.unitypackage
  7. Inside the folder Assets/Parse/Plugins remove everything (Unity.Compat.dll and Unity.Tasks.dll) from the root and keep just the folder dotNet45

2. Firebase Android setup

  • Switch platform to Android
  • Go to Assets -> External Dependency Manager -> Android Resolver -> Settings
  • Make sure "User Jetifier" is deactivated
  • Run Assets -> External Dependency Manager -> Android Resolver -> Force Resolve
  • Inside Unity, go to Menu Achroma -> Copy Google Sign In Files. This is needed to to make Google Sign In work.

3. Project setup

  • Add the Achroma Plugin to the project by adding it by git URL: https://github.com/Realm-Runner-Studios/Achroma-Unity-Plugin.git
  • Go to Edit -> Project Settings. On the left side, under "Services", select "In-App Purchasing" and click on the switch button on the top right corner

3.1 Firebase setup

In Firebase, make sure that under "Remote Config" there's a parameter with the name of the app + "LastVersion" + platform. For example, if the bundle identifier on Android is com.realmrunnerstudios.achromaAdenture, a parameter must be third element of the bundle identifier (achromaAdventure) + "LastVersionAndroid". The result would be "achromaAdventureLastVersionAndroid" For iOS, if the bundle identifier is "com.realmrunnerstudios.achroma-adventure", the result would "achromadventureLastVersioniOS" (the hiphen is not accepted on Firebase)

Do the same for the "LastVersion" parameters. You must also add two parameters for the app store address, like: achromaAdventureLinkStoreAndroid achromaadentureLinkStoreiOS These parameters will contain a link to the App Store or Google Play store where the user will be taken to to download a new version.

4. Usage

To use the plugin, add using AchromaAPI; to the head of your script.

This plugin has areas of usage depending on what you need todo:

AchromaInstanceAssets for anything related to Assets like Characters, Locations, Objects and Actions.

AchromaInstanceAuth for anything related to Sign In, Sign Out

AchromaInstanceAchievements for anything related to user's achievements

AchromaInstanceDatabase for anything related to the user's account and their data

AchromaInstanceMatch for anything related to Companion match

AchromaInstanceShards for anything related to Shards Animations

AchromaInstanceStory for anything related to the Adventure Mode

AchromaInstanceUI for anything related to UI elements standardized for Achroma

You can use the plugin as follows. Everything starts with the AchromaAPI singleton:

public void SignIn()
{
AchromaAPI.AchromaInstanceAuth.SignIn(false, true, delegate (bool success, string message)
{
if (success)
{
AchromaAPI.AddUserNotification("Logged in");
}
else
{
AchromaAPI.ShowErrorMessage(message);
}
});
}

Android build

When building for Android, make sure a folder with the name FirebaseApp.androidlib is created inside Assets/Plugins/Android. If it's not created, run Assets -> External Dependency Manager -> Android Resolver -> Forse Resolve

iOS build

When building for iOS, inside Xcode make sure Bitcode is disabled for the UnityFramework package
